# 阿里云盘Web管理界面

这是一个为阿里云盘CLI客户端提供的Web管理界面，让你可以通过浏览器来管理阿里云盘中的文件。

## 功能特性

- 📁 **文件浏览**: 浏览云盘中的文件和文件夹
- 📤 **文件上传**: 支持单文件和多文件上传
- 📥 **文件下载**: 获取文件下载链接
- 🗂️ **文件夹管理**: 创建新文件夹
- ✏️ **文件重命名**: 重命名文件和文件夹
- 🗑️ **文件删除**: 删除文件和文件夹
- 📊 **存储统计**: 显示存储空间使用情况
- 🎨 **现代化界面**: 美观的响应式设计

## 系统要求

- Go 1.22 或更高版本
- 已安装并配置的阿里云盘CLI客户端
- 已登录阿里云盘账号

## 安装和运行

### 1. 确保已登录

首先确保你已经使用命令行客户端登录了阿里云盘：

```bash
./aliyunpan login
```

### 2. 安装依赖

在项目根目录运行：

```bash
go mod tidy
```

### 3. 启动Web服务器

#### 方法一：使用启动脚本（推荐）

```bash
./web/start.sh
```

#### 方法二：手动启动

```bash
cd web
go run main.go
```

### 4. 访问Web界面

打开浏览器访问：http://localhost:8080

## 使用说明

### 基本操作

1. **浏览文件**: 点击文件夹名称进入目录，使用面包屑导航返回上级目录
2. **上传文件**: 点击"上传文件"按钮，选择要上传的文件
3. **下载文件**: 点击文件右侧的"下载"按钮
4. **新建文件夹**: 点击"新建文件夹"按钮，输入文件夹名称
5. **重命名**: 点击"重命名"按钮，输入新名称
6. **删除**: 点击"删除"按钮，确认删除操作

### 界面说明

- **顶部导航**: 显示当前用户信息和版本号
- **存储统计**: 显示云盘存储空间使用情况
- **工具栏**: 提供常用操作按钮
- **面包屑导航**: 显示当前路径，支持快速导航
- **文件列表**: 显示当前目录下的所有文件和文件夹

### 文件操作

- **文件夹**: 显示为📁图标，点击可进入
- **文件**: 显示为📄图标，支持下载操作
- **文件信息**: 显示文件大小和修改时间
- **操作按钮**: 每个文件/文件夹右侧都有操作按钮

## 配置选项

### 环境变量

- `PORT`: 设置Web服务器端口（默认：8080）
- `ALIYUNPAN_CONFIG_DIR`: 设置配置文件目录

### 示例

```bash
# 使用自定义端口启动
PORT=9000 ./web/start.sh

# 使用自定义配置目录
ALIYUNPAN_CONFIG_DIR=/path/to/config ./web/start.sh
```

## API接口

Web界面提供以下REST API接口：

### 文件管理

- `GET /api/files?path=<path>` - 获取文件列表
- `POST /api/upload` - 上传文件
- `GET /api/download?fileId=<fileId>` - 获取下载链接
- `DELETE /api/file?fileId=<fileId>` - 删除文件
- `POST /api/folder` - 创建文件夹
- `PUT /api/rename` - 重命名文件
- `PUT /api/move` - 移动文件

### 用户信息

- `GET /api/user` - 获取用户信息
- `GET /api/quota` - 获取存储配额信息

## 故障排除

### 常见问题

1. **启动失败**: 确保已安装Go环境并运行了`go mod tidy`
2. **登录失败**: 确保已使用命令行客户端登录
3. **端口被占用**: 使用`PORT`环境变量指定其他端口
4. **权限问题**: 确保有读取配置文件的权限

### 日志查看

Web服务器会在控制台输出详细的日志信息，包括：
- 启动信息
- 用户登录状态
- API请求日志
- 错误信息

## 开发说明

### 项目结构

```
web/
├── main.go          # Web服务器主文件
├── start.sh         # 启动脚本
```

### 技术栈

- **API**: RESTful API设计
- **认证**: 复用阿里云盘CLI的登录状态

如需添加新功能，可以：

1. 在`main.go`中添加新的API接口
2. 在前端HTML中添加对应的UI组件
3. 在JavaScript中添加交互逻辑
